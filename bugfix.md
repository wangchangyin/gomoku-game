# Gomoku Game Bugfix 文档

## 项目存在的问题

### 1. 胜负判断算法效率低
- **问题描述**: 当前 `checkWinner` 函数遍历整个棋盘来检查获胜，时间复杂度为 O(n²)
- **影响**: 棋盘越大，性能越差
- **建议优化**: 改为从落子位置四周检查，时间复杂度降为 O(1)

### 2. 缺少禁手规则
- **问题描述**: 标准五子棋黑棋有三三禁手、四四禁手、长连禁手规则，当前实现未包含
- **影响**: 不符合专业五子棋规则
- **建议**: 添加禁手检测逻辑

### 3. 无撤销（悔棋）功能
- **问题描述**: 用户无法撤销上一步操作
- **影响**: 用户体验不佳，误操作无法恢复
- **建议**: 添加历史记录栈，支持悔棋功能

### 4. 无计时功能
- **问题描述**: 缺少双方步时/总用时统计
- **影响**: 无法用于正式比赛场景
- **建议**: 添加计时器组件

### 5. 魔法数字硬编码
- **问题描述**: `BOARD_SIZE = 15` 等常量硬编码在多处
- **影响**: 维护困难，修改棋盘大小需要改多个文件
- **建议**: 统一配置到 constants.ts 文件

### 6. 缺少单元测试
- **问题描述**: 核心逻辑（胜负判断、平局检测）无单元测试
- **影响**: 代码质量无法保证，回归风险高
- **建议**: 添加 vitest + @testing-library/react 测试

### 7. 无错误边界处理
- **问题描述**: 缺少 Error Boundary 组件
- **影响**: 运行时错误可能导致整个应用崩溃
- **建议**: 添加 React Error Boundary

### 8. 移动端适配问题
- **问题描述**: 棋盘在小屏幕上可能溢出
- **影响**: 移动端用户体验差
- **建议**: 添加响应式布局，棋盘自适应缩放

### 9. 无落子标记
- **问题描述**: 不显示最后一步位置
- **影响**: 难以跟上对局节奏
- **建议**: 添加最后一步高亮标记

### 10. 缺少音效和动画
- **问题描述**: 落子、获胜无视觉/听觉反馈
- **影响**: 游戏体验单调
- **建议**: 添加落子音效、获胜动画
